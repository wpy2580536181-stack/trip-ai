## 残留问题清单

### 数据层

| 问题 | 影响 | 建议方案 | 复杂度 |
|------|------|---------|--------|
| **消息增量持久化静默吞错** | DB 写入失败时用户下次看不到完整消息 | `persistAssistant` catch 改为一轮重试 + 最终告警 | 低 |
| **compressConversation 与主流程写同一条记录** | 可能出现文本覆盖 | 压缩只压缩旧消息（< assistantMsgId），不碰正在写的消息 | 中 |
| **进程崩溃未 flush 的内容丢失** | 3s 窗口内的增量丢失 | persistent queue / 日志 WAL | 高 |
| **无全局错误处理器** | 未捕获异常导致进程退出 | `app.use((err, req, res, next) => ...)` 兜底 | 低 |

### LLM 调用层

| 问题 | 影响 | 建议方案 | 复杂度 |
|------|------|---------|--------|
| **res.write 失败后 Agent 不感知** | 连接已断但 Agent 继续运行 | heartbeat interval 中检测 `isClientConnected()`，失败则 `abortController.abort()` | 低 |

### 前端层

| 问题 | 影响 | 建议方案 | 复杂度 |
|------|------|---------|--------|
| **fetch 超时 60s 但流式连接可能持续更久** | 60s 后自动断开 | 针对 SSE 请求移除 timeout 或设更大值（5min） | 低 |
| **无连接重试** | 网络波动导致流中断，用户需重新发送 | fetch 失败时自动重连（需服务端支持续接） | 高 |
| **无错误状态展示** | 用户不知是服务器慢还是断了 | 心跳 >10s 未收到显示"连接不稳定"提示 | 低 |
