实施路线图

第一阶段：搭架子
核心变更
  • 安装 Naive UI + 字体图标库
  • 创建布局组件 AppLayout.vue（侧边栏 + 内容区）
  • 替换 App.vue 底栏为侧边栏布局
  • 配置 Naive UI 主题系统（light/dark）
  • 定义全局 CSS 变量（暖灰色板）
  • 更新 main.ts 接入 Naive UI
第二阶段：改页面
逐个替换
  • 游客页面：Login / Register / ResetPassword
  • 核心页面：Home / Chat / Detail / History
  • 个人页面：Profile / TokenUsage / About
  • 管理页面：Knowledge / Feedback / Trace / Architecture
第三阶段：打磨
细节优化
  • 暗色模式切换动画
  • 页面过渡动画
  • 侧边栏折叠/展开
  • 删除 Vant 依赖
  • 清理全局样式冗余
  • 响应式断点微调

关键文件变更清单

新增：
+ src/components/layout/AppLayout.vue
+ src/components/layout/Sidebar.vue
+ src/styles/theme.ts （Naive UI 主题配置）
修改：
~ src/main.ts（接入 Naive UI）
~ src/App.vue（替换布局）
~ src/router/index.ts（布局嵌套路由）
~ src/views/*.vue（14 个页面逐个替换）
~ src/styles/common.css（精简为全局极简样式）
删除：
- vant 包依赖
- unplugin-vue-components VantResolver
- 所有 van-* 组件引用

关键决策：第一阶段结束时，Vant 仍然在依赖中（页面未改完时保留）。第二阶段逐页替换，第三阶段再移除 Vant。这样任何一步出问题都可以回退。
